Architecture: Event-Driven Modular Design

5 independent modules communicating through a central message bus

Keybindings & navigation
How they communicate:
• Each module publishes events and subscribes to events it cares about
• APIClient publishes SearchResultsEvent → UI subscribes to display them
• UI publishes PlayRequestEvent → Player subscribes to play
• Player publishes TrackChangedEvent → UI subscribes to update display
• Auth publishes AuthReadyEvent → API subscribes to start making calls

Pros

  • Each module independently testable
  • Subagents can build in parallel
  • New features = new event handlers

Trade-offs

  • Event contracts must be defined early
  • Slightly more boilerplate than monolithic
